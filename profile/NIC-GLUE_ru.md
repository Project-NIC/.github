# NIC-GLUE (IN / OUT) — швы

*Тизер · для Show HN, r/embedded, Hackaday и подобных сообществ.*

---

Библиотеки NIC намеренно ничего не знают друг о друге: MLA хранит непрозрачные
байты, DMD пакует пакеты фиксированной ширины, KSF преобразует байты — ни одна не
подозревает о существовании других. *Клей* — это любой код, выравнивающий швы, и
выровнять их правильно — это и есть вся работа.

**GLUE-IN** связывает сторону записи: строку с датчика, опциональное сжатие DMD, в
контейнер MLA — выставляя бит «compressed» и расстояние до keyframe так, чтобы
каждый ротированный файл декодировался сам по себе. **GLUE-OUT** — та же идея
наоборот: проходит готовый контейнер обратно наружу в CSV или SQLite, попутно
распаковывая.

Ни то, ни другое — не фреймворк, который нужно принять. Это рабочий пример плюс
небольшой каталог той горстки швов, где библиотеки должны согласоваться —
записанный однажды, чтобы никому не пришлось открывать его заново.
